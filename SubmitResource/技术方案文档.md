# AI Stock Insight — 技术方案文档
## 基于"投资的道与术"哲学的四层多智能体博弈分析系统

---

> **项目名称**: AI Stock Insight  
> **核心理念**: 以"阴阳归因"为哲学内核，用多智能体博弈替代单点决策，实现从数据到投资决策的全链路自动化推理。

---

## 一、项目概述

### 1.1 问题定义

散户投资者面临三大核心困境：

1. **信息不对称**：在数据挖掘、量化建模能力上全面落后于机构；
2. **单维度决策**：习惯只看技术面或只看新闻，缺乏多维交叉验证；
3. **情绪化操作**：缺乏独立的"反向意见"制衡，容易追涨杀跌。

### 1.2 解决方案

AI Stock Insight 构建了一套**四层十二智能体博弈系统**，将机构级的多团队分析流程完整数字化：

- 替代"分析师团队"：5个专业 Agent 并行深度研究；
- 替代"研究员辩论"：多空双方进行结构化博弈辩论；
- 替代"交易员决策"：基于量化评分矩阵制定交易策略；
- 替代"风险委员会"：3种风险偏好同步评估，Portfolio Manager 综合裁决。

## 三、核心创新点（技术维度 · 创新性）

### 3.1 「阴阳归因」哲学框架

本系统将《投资的道与术》中的投资哲学体系工程化，构建了**「阴阳归因」智能决策框架**：

```
Portfolio Manager 第零步 — 阴阳归因

    ├── 阳（价值驱动）→ 权重调整: 基本面 × 1.5
    │    └── 价值投资策略：ROE稳定性是核心判据，适合长期持有
    │
    ├── 阴（预期驱动）→ 权重调整: 情绪/新闻 × 1.3
    │    ├── 事件驱动策略：严格止损，催化剂消退即退出
    │    └── 透支预期检测：涨幅 vs 业绩增速，超出触发警示
    │
    ├── 守阴待阳 → 成长股：判断「预期转化为价值」的概率
    │
    ├── 守阳待阴 → 逆向投资：安全边际 > 30% 才触发买入
    │
    └── 阴极反转 → 保守派一票否决，强制0%仓位
```

这一框架使系统不仅是数据聚合工具，而是具有哲学视角的**因果推理引擎**。

### 3.2 动态辩论博弈机制

区别于传统单 Agent 分析，系统使用**评分差触发 → 多轮辩论 → 收敛退出**自适应协议：

```python
# 辩论触发逻辑（实际源码片段）
score_diff = abs(bullish.score - bearish.score)
if score_diff >= debate_threshold:
    for round_num in range(max_debate_rounds):
        bullish = update_with_rebuttal(bullish)
        bearish = update_with_rebuttal(bearish)
        new_diff = abs(bullish.score - bearish.score)
        if new_diff < debate_threshold * 0.5:
            break  # 评分收敛，提前退出
```

- 分歧小 → 直接决策，节省时间；
- 分歧大 → 多轮博弈直至收敛，保证决策质量；
- 用户可配置辩论阈值（1-10）和最大轮次（1-5）。

### 3.3 A股特色数据融合

专为中国本土市场设计，融入两个机构投资者核心信号：

| 数据源 | 投资含义 | 在系统中的作用 |
|--------|----------|----------------|
| 北向资金（陆股通） | 外资/QFII实时认可度 | 验证基本面评估的「外资视角」 |
| 龙虎榜游资 | 主力资金介入与撤离信号 | 量化信号置信度修正 |

这是专为 A 股市场定制的创新能力，通用 AI 工具完全缺失此维度。

### 3.4 「透支预期一票否决」机制

```
若检测到：股价涨幅 >> 营收/利润增速
         + 市场情绪极度乐观

则触发链路：
  空头研究员识别「阴极反转」
  → 保守派强制 0% 仓位（一票否决）
  → Portfolio Manager 激活「阴极反转」分支
  → 最终输出：卖出/规避
```

在估值泡沫情形下大幅提升系统的风险识别鲁棒性。

## 四、效率设计

### 4.1 并行化执行

Layer 1 的 5 个 Agent **完全并行**，实际执行耗时 ≈ 最慢单 Agent 耗时（而非5倍串行）：

```python
tasks = [
    run_analyst("fundamentals_analyst", system._run_fundamentals_analyst),
    run_analyst("sentiment_analyst",    system._run_sentiment_analyst),
    run_analyst("news_analyst",         system._run_news_analyst),
    run_analyst("technical_analyst",    system._run_technical_analyst),
    run_analyst("quant_analyst",        system._run_quant_analyst),
]
for completed_task in asyncio.as_completed(tasks):
    role_name, res = await completed_task
    yield stream_event(role_name, res)  # 完成即刻推送前端
```

- Layer1 效率提升：理论 **4-5×** 相比串行；
- 多股对比：N只股票完全并行，每完成一只立刻展示进度。

### 4.2 NDJSON 流式传输协议

```
Client ←─── HTTP Streaming ──── Server

{"type":"status","message":"初始化..."}
{"type":"agent_output","role":"fundamentals_analyst","data":{...}}
{"type":"agent_output","role":"technical_analyst","data":{...}}
{"type":"layer_start","layer":2,...}
{"type":"final_result","data":{"recommendation":"买入",...}}
```

用户**首屏看到内容的延迟 ≈ 最快 Agent 的单次耗时**，渐进式展示极大提升体验。

## 五、鲁棒性设计

### 5.1 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| 数据接口超时/失败 | try/except 降级，返回限定提示文本，不中断流程 |
| LLM 输出幻觉分数 | 多重正则兜底，异常时回退中位数评分 5.0 |
| 多股对比某股失败 | 单股失败独立上报，其余股票照常展示 |
| 前端 JSON 解析失败 | catch 静默跳过，输出 warning 而非崩溃 |
| 代理/VPN 网络限制 | 新浪财经 API 绕过代理取股票名称 |

### 5.2 噪音信号过滤

- **情绪噪音**：情绪分析师提示词明确设定"极端情绪为反向指标"，不被放大；
- **新闻噪音**：区分"短期噪音"与"长期趋势"，短期事件不上升权重；
- **数据噪音**：量化分析师使用3年均值平滑单季度异常值。

### 7.1 为散户提供机构级分析能力

| 能力维度 | 传统散户 | 本系统 |
|----------|----------|--------|
| 基本面分析 | 手动查报表 | 自动获取 + AI 深度解读 |

| 多空博弈模拟 | 完全没有 | 结构化辩论机制 |
| 风险评估 | 模糊感觉 | 3种偏好量化仓位建议 |
| 投资哲学 | 个人经验 | 阴阳归因框架指导 |

### 7.2 多股横向择优（竞品选股核心功能）

- 支持最多5只股票并行分析，按综合评分自动排名；
- 输出：综合评分、各维度细分分数、策略类型归因、建议仓位；
- 每完成一只立刻推送进度，用户无需等待全部完成。

### 7.3 教育与认知价值

系统的每一步分析**透明可追溯**：用户不仅能看到"买入"结论，更能看到5个专业角度的完整推理过程以及多空双方的辩论细节，真正培养投资认知而非依赖黑盒。

## 八、人机交互设计

### 8.1 零门槛输入

- 单股分析：输入6位股票代码，即刻开始；
- 多股对比：输入2-5个代码，自动并行并排名；
- 参数持久化：API Key 和模型选择保存在 `localStorage`。

### 8.2 渐进式可解释展示

前端时间轴按四层逐级展开，分析师结果实时渲染：

```
Layer 1 ─── [基本面✅] [情绪✅] [新闻✅] [技术✅] [量化✅]
              ↓ 每完成一个立即显示
Layer 2 ─── [多空辩论 · 双栏展示多空分歧]
              ↓
Layer 3 ─── [交易员决策 · 仓位区间 · 止损目标]
              ↓
Layer 4 ─── [3种风险偏好仓位建议] + [最终投资决策]
```

### 8.3 参数可调节性

| 参数 | 含义 | 默认 |
|------|------|------|
| `debate_threshold` | 触发辩论的多空评分差阈值 | 3.0 |
| `max_debate_rounds` | 最多辩论轮次 | 2 |
| `model` | 底层 LLM 选型 | Qwen2.5-7B |
| `api_key` / `base_url` | 自定义推理节点 | SiliconFlow |
