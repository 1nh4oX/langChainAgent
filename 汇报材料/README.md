## 🚀 快速开始（准备流程）

### 第1步：通读材料（30分钟）

1. **先看**：`演讲要点-快速记忆版.md`
   - 了解整体结构（220秒流程）
   - 记住关键词和核心概念
   - 标记不熟悉的部分

2. **再看**：`演讲稿.md`
   - 完整读2遍（不用背诵）
   - 理解每段的逻辑和目的
   - 用自己的话复述一遍

3. **最后看**：`PPT大纲.md` 和 `PPT代码片段.md`
   - 确认 PPT 内容覆盖要点
   - 检查代码片段是否清晰

---

### 第2步：制作 PPT（1-2小时）

#### 方案A：从零制作（推荐）⭐

**工具：** PowerPoint / Keynote / Google Slides

**模板建议：**
- 简洁风格（不要花里胡哨）
- 深色背景（黑/深蓝/深灰）+ 白色文字
- 代码用浅色背景（提高对比度）

**制作步骤：**

1. **创建7页 PPT**（按 `PPT大纲.md` 的逻辑顺序）

2. **设计标题页**（第1页）
   - 标题：智能股票分析 Agent
   - 副标题：基于 LangChain 的 A 股分析系统
   - 你的名字 + 日期

3. **绘制整体流程图**（第2页）
   ```
   用户查询
       ↓
   Streamlit UI（前端展示）
       ↓
   Agent 初始化（绑定工具 + 提示词）
       ↓
   Agent 循环执行（ReAct模式）
       ↓
   工具调用（获取真实数据）
       ↓
   数据返回 → AI 分析 → 输出报告
   ```
   - 标注三层架构：展示层、Agent层、工具层

4. **插入工具实现代码**（第3页）
   - 从 `PPT代码片段.md` 复制**完整工具代码**
   - 使用 [Carbon](https://carbon.now.sh) 生成代码截图
   - 或使用 PPT 代码块（Consolas 字体，16-18pt）
   - **关键行用红色高亮**（@tool装饰器、返回Markdown）
   - 列出5个工具名称

5. **插入Agent搭建代码**（第4页）
   - 复制**完整Agent初始化代码**
   - 高亮关键操作：bind_tools、Prompt构建、Runnable链
   - 展示系统提示词（SYSTEM_PROMPT）

6. **插入Agent运行代码**（第5页）
   - 复制**完整循环代码**
   - 高亮关键步骤：检查tool_calls、执行工具、递归调用
   - 添加执行流程示例（3轮调用过程）

7. **插入前端代码**（第6页）
   - 复制Streamlit UI核心代码
   - 或使用截图展示界面效果

8. **总结页**（第7页）
   - 4个创新点（大字）
   - 技术栈：LangChain + AkShare + Streamlit
   - 代码量：700行
   - 功能：5工具 + Agent + UI

8. **Q&A页**（第10页，备用）
   - 列出可能的问题
   - 简短答案（1-2句话）

### 第3步：练习演讲（1小时）

#### 第1轮：通读（15分钟）

- 对着 PPT 读演讲稿
- 不计时，先把流程走通
- 标记卡壳的地方

#### 第2轮：计时（15分钟）

- 设置计时器
- 完整讲一遍
- 记录实际用时（目标：5分50秒）

**如果超时：**
- 工具实现：80s → 60s（只讲一个完整例子）
- Agent搭建：70s → 60s（简化代码展示）
- Agent运行：90s → 70s（简化流程示例）
- 前端展示：40s → 30s（快速带过）
- **压缩后总时长：约4分30秒**

**如果时间太短：**
- 每段代码多解释1-2句
- 增加手势和停顿
- 放慢语速
- 详细讲解工具列表（5个工具）

#### 第3轮：脱稿（30分钟）

- 只看 PPT，不看演讲稿
- 用自己的话讲
- 允许卡壳，但不要停下来

**技巧：**
- 记住每页的**关键词**（不是全文）
- 用**"总分总"结构**：先说是什么，再说怎么做，最后总结
- 代码部分可以看屏幕读（不算脱稿）

#### 第4轮：录像（5-10分钟，可选）

- 用手机录下自己讲
- 回看录像，找问题：
  - 语速是否合适？
  - 有没有口头禅？（"然后"、"嗯"、"就是"）
  - 手势是否自然？
  - 眼神是否呆板？

## 💡 高频问题 FAQ

### Q1: 时间不够，讲不完怎么办？

**A:** 优先保留这3部分：
1. ⭐ 代码1 - Agent架构（bind_tools）
2. ⭐ 代码2 - Agent循环（ReAct）
3. ⭐ 代码3 - 工具实现（@tool）

其他部分可以快速带过或跳过。

---

### Q2: 被问住了，答不上来怎么办？

**A:** 万能回答公式：

> "这是个好问题。（停顿1秒）从我们的实现来看，（看PPT/代码）主要是通过...（说你知道的部分）。如果您感兴趣，我们可以会后详细讨论。"

**关键：** 不要慌，诚实最重要。如果真的不知道，就说"这个我需要再确认一下"。

---

### Q3: 演讲时紧张怎么办？

**A:** 三个技巧：

1. **深呼吸** - 开场前深呼吸3次
2. **眼神交流** - 找几个友好的听众，看他们的眼睛
3. **慢下来** - 紧张时语速会加快，刻意放慢

---

### Q4: 忘词了怎么办？

**A:** 看 PPT！PPT 就是你的提词器。

**万能过渡句：**
- "接下来我们看..."（指向PPT）
- "这部分的核心是..."（读PPT标题）
- "具体来说就是..."（读代码注释）

---

### Q5: 代码太复杂，讲不清楚？

**A:** 用类比！

- **bind_tools** → "就像给AI配了一个工具箱，告诉它'你可以用这些工具'"
- **递归循环** → "就像打游戏，每一轮做决策，用道具，再看结果，循环往复"
- **@tool装饰器** → "就像给函数贴了个标签，自动变成工具"
